How is this Blog Made?

2014-12-19 11:59

Comments

It was a long time that I was wondering about making a blog. in another hand I had alot of expectations. this blog shoud have a simple looking. I prefered single culomn design. it should support bilingual. I also wanted features like adding images,video,sounclips,code highlighting and commenting. the first thing that came to my mind was using famous CMS like Wordpress. Wordpress could provide all of my needs. a friend of mine introduced me to website about wordpress. I also took a look at TemplateMonster for finding a template that answer my needs. but i wasn't sattisfied with any of them. I was drown in my thoughts when google bring me to a very cool project called nikola.

What is The diffrence between nikola and CMS like Joomla and Wordpress?

For Answering this question, we have to determinie diffrences between static and dynamic Websites. In a Dynamic website contents are stored in a database. When a request is recieved by the server. contents related to that request is read from database and then will be proccessed by another Code and will be converted to HTML code ready for sending to the User.this means alot of proccess should be done per each single request.

In static Sites HTML code is prepared before and it will be sent immidiately after getting the request.

Nikola produces Static Webpages while Joomal,Wordpress produce Dynamic ones.

This project is written in python. its input is a text file with an special structure and its output is a series of HTML pages containig all pages of the blog.

If we make a general comparison between static and Dynamic pages, we will have:

  • in first look static pages load faster than Dynamic one, its reason is completely obvious. less proccess.
  • less complextiy because of no need to database and plugins for adding new functionalities such as code Highlighting
  • in static pages security is near 100%. because HTML pages are made before and don't change.in this case the only vulnerabilite can be webserver.modern webservers are solid as rock. it's nearly impossible to penetrate them from remote.
  • in Dynamic websites because some data from the user is sent to database. if this data be malicious can put your site in a risk to attacks like SQL injection
  • another risk in Dynamic Pages is vulnerabilities in Plugins which is used for most attacks to websites.specialy in cases that an exclusive plugin is written,and because of incompatibiltiy between this plugin and CMS version. your unable to upgrade your CMS.

by considering reasons mentioned above, I got that static site is the best solution for me. why ?

  • more speed
  • less complexity
  • more secure
  • low cost (I only paid for Domain registartion)

now i made my decison to use static page, it's time to talk about hosting. if i use CMS I have to pay a company for hosting my site. there are alot of service with different costs.but i'm not using CMS. Nikola itself suggest GithubPages.

For making a github page you have to make a repository in github. then push your content in that repo.after several minutes you can surf your site (The first time you push to Your repo it takes 30 min until your content be accessible. after that it will be instanous). In this mode your site become a subdomain under github.io

if you want to use your domain name make a file named CNAME in your repo containing your domain name.

now My site is up. it's time to talk about commenting system. I use a service called disqus. it's very simple and comptible with nikola. In last we need a reporting system i am using googleAnalytics right now.

if you was intrested using nikola ,I recommend take a look at Cheatsheet on it's website.it contains everything you need to know. if you have any question you can email me on whiletruebefree at gmail
